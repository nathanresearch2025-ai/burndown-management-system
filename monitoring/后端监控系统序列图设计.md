# 后端监控系统序列图设计

本文给出监控方案核心时序：
1. 指标采集与可视化流程
2. 告警触发与通知流程
3. 异常与降级处理流程

---

## 1. 指标采集与可视化时序图

```mermaid
sequenceDiagram
  autonumber
  participant APP as Java Backend\nActuator+Micrometer
  participant PRO as Prometheus
  participant GRA as Grafana
  participant USER as 开发/测试/管理员

  PRO->>APP: 周期抓取 /actuator/prometheus (15s)
  APP-->>PRO: 返回 JVM/HTTP/Process/DB 指标
  PRO->>PRO: 写入 TSDB

  USER->>GRA: 打开监控看板
  GRA->>PRO: PromQL 查询（时间窗口+实例维度）
  PRO-->>GRA: 返回时序数据
  GRA-->>USER: 展示实时图表（CPU/内存/JVM/访问量）

  loop 自动刷新
    GRA->>PRO: 周期查询（10s~30s）
    PRO-->>GRA: 增量数据
    GRA-->>USER: 刷新图表
  end
```

## 2. 告警触发与通知时序图

```mermaid
sequenceDiagram
  autonumber
  participant PRO as Prometheus
  participant AR as Alert Rule Engine
  participant AM as Alertmanager
  participant CH as 通知渠道\n(邮件/企业微信/钉钉)
  participant ONCALL as 值班人员
  participant GRA as Grafana

  PRO->>AR: 周期执行规则评估
  AR->>AR: 判断阈值与持续时间(for)

  alt 触发告警
    AR-->>AM: 发送告警事件(firing)
    AM->>AM: 分组/抑制/路由
    AM-->>CH: 发送告警通知
    CH-->>ONCALL: 推送告警消息

    ONCALL->>GRA: 打开告警看板排查
    GRA->>PRO: 查询关联指标趋势
    PRO-->>GRA: 返回异常时段数据
    GRA-->>ONCALL: 展示根因线索
  else 告警恢复
    AR-->>AM: 发送恢复事件(resolved)
    AM-->>CH: 发送恢复通知
    CH-->>ONCALL: 已恢复
  end
```

## 3. 异常与降级处理时序图

```mermaid
sequenceDiagram
  autonumber
  participant APP as Java Backend
  participant PRO as Prometheus
  participant GRA as Grafana
  participant USER as 用户

  alt 应用指标端点异常
    PRO->>APP: 抓取 /actuator/prometheus
    APP--x PRO: 超时/5xx
    PRO->>PRO: 标记 target down
    PRO-->>GRA: 数据缺失/实例离线状态
    GRA-->>USER: 显示实例异常提示
  else Prometheus 短暂不可用
    USER->>GRA: 查询监控图表
    GRA->>PRO: 查询请求
    PRO--x GRA: 不可达
    GRA-->>USER: 展示“数据源不可用”降级提示
  else 单指标异常抖动
    PRO->>PRO: 规则评估
    PRO->>PRO: 因 for 窗口未满足，不触发告警
    PRO-->>GRA: 正常展示波动曲线
    GRA-->>USER: 建议持续观察
  end
```

---

## 4. 时序设计说明

1. **拉模型采集**：Prometheus 主动抓取，避免应用侧推送复杂度。
2. **规则先评估后通知**：告警由规则引擎统一判定，减少误报。
3. **告警降噪机制**：依赖 `for`、分组、抑制、静默，避免通知风暴。
4. **可视化与排障闭环**：告警触发后可快速进入 Grafana 下钻定位。
5. **降级可见性**：采集失败、数据源不可用均需在看板可见，避免“静默故障”。
