# CompletableFuture 并行查询测试用例（简版）

## 1. 目标
验证并行拉取“进行中任务 + 燃尽图数据”后，站会问答流程的稳定性与正确性。

---

## 2. 基础场景
### 用例 1：正常数据（任务 + 燃尽图均有数据）
- **前置条件**：Sprint 中存在进行中任务，燃尽点数据完整
- **输入**：项目 ID = 1001，Sprint ID = 302
- **预期**：
  - 两个异步任务均成功返回
  - 最终摘要包含任务概况与燃尽风险
  - 响应时间小于原串行方案

### 用例 2：只有任务数据（燃尽图为空）
- **前置条件**：进行中任务存在，但没有燃尽点
- **输入**：项目 ID = 1001，Sprint ID = 999
- **预期**：
  - 任务数据返回正常
  - 燃尽图工具返回“无数据”提示
  - 摘要中明确提示燃尽图缺失

### 用例 3：只有燃尽图数据（任务为空）
- **前置条件**：无进行中任务，但有燃尽点
- **输入**：项目 ID = 1002，Sprint ID = 401
- **预期**：
  - 任务返回空提示
  - 燃尽图正常返回
  - 摘要包含风险评估，但说明任务为空

---

## 3. 异常场景
### 用例 4：任务接口失败
- **前置条件**：任务工具抛出异常
- **预期**：
  - 并行任务中任务侧失败
  - 仍可返回燃尽图摘要
  - 返回结果包含降级提示

### 用例 5：燃尽图接口失败
- **前置条件**：燃尽图工具抛出异常
- **预期**：
  - 任务数据正常
  - 风险评估缺失
  - 返回结果提示燃尽图不可用

### 用例 6：两个接口同时失败
- **前置条件**：任务与燃尽图工具都失败
- **预期**：
  - 返回统一的失败提示
  - 不崩溃

---

## 4. 性能场景
### 用例 7：并行性能验证
- **前置条件**：任务接口耗时 ~300ms，燃尽图耗时 ~400ms
- **预期**：
  - 总耗时接近 max(300ms, 400ms) + LLM 时间
  - 明显优于串行方案

---

## 5. 备注
如果后续加入风险评估并行调用，可扩展更多场景验证。
