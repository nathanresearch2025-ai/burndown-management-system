## 测试案例 1: 基本功能测试

### 目标
验证 AI 生成描述的基本流程是否正常工作

### 测试步骤

1. **进入任务看板**
   - 点击左侧菜单"任务看板"
   - 选择一个活跃的冲刺

2. **创建新任务**
   - 点击"创建任务"按钮
   - 填写以下信息：
     - 项目: 选择任意项目
     - 冲刺: 选择当前冲刺
     - 标题: `优化用户登录性能`
     - 类型: `任务`
     - 优先级: `高`
     - 故事点: `5`

3. **生成 AI 描述**
   - 点击描述框上方的 "✨ AI 生成描述" 按钮
   - 观察按钮状态变化（应显示 Loading）
   - 等待 2-3 秒

4. **验证结果**
   - ✅ 描述框自动填充了生成的内容
   - ✅ 生成的描述包含任务目标、实现要点等
   - ✅ 如有相似任务，底部显示参考提示
   - ✅ 可以编辑生成的描述

5. **保存任务**
   - 点击"确定"按钮
   - 验证任务成功创建

### 预期结果

- 生成时间: < 3 秒
- 描述内容: 包含结构化的任务说明
- 相似任务: 显示 0-5 个相关历史任务

## API 测试（可选）

### 使用 curl 测试 API

**重要提示：** AI 生成接口需要认证，必须先登录获取 JWT token。

```bash
# 1. 登录获取 token
curl -X POST http://159.75.202.106:30080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "password123"
  }'

# 响应示例：
# {
#   "token": "eyJhbGciOiJIUzI1NiJ9...",
#   "username": "admin",
#   "permissions": ["USER_READ", "USER_WRITE", ...]
# }

# 2. 使用 token 调用 AI 生成接口（将 YOUR_TOKEN_HERE 替换为实际 token）
curl -X POST http://159.75.202.106:30080/api/v1/tasks/ai/generate-description \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -d '{
    "projectId": 1,
    "title": "优化用户登录性能",
    "type": "TASK",
    "priority": "HIGH",
    "storyPoints": 5
  }'
```

### 预期响应

```json
{
  "description": "生成的任务描述内容...",
  "generatedBy": "external-llm",
  "similarTasks": [
    {
      "taskKey": "PROJ-123",
      "title": "用户登录功能",
      "similarity": 0.85
    }
  ]
}
```

2. **降级模式**: 如果 AI 服务未配置或不可用，系统会自动使用降级模式，返回基础模板

3. **测试数据**: 系统已初始化测试数据，包括用户、项目、冲刺和历史任务

4. **速率限制**: 注意每小时只能生成 10 次，测试时请合理安排

5. **缓存机制**: 相同标题和类型的任务在 24 小时内会返回缓存结果
