# 需求文档（简版）

## 1. 项目背景
燃尽图管理系统已是 Spring Boot 后端，现需接入 Agent 能力，实现“需求→规划→工具调用→记忆→多 Agent 协作→部署→监控”的全链路，并支持自然语言查询与摘要生成。

## 2. 目标
- 支持用户自然语言输入并返回结构化业务摘要
- 支持多轮对话记忆（上下文延续）
- 支持工具调用（任务、燃尽图、风险评估）
- 支持监控与可观测性

## 3. 典型使用场景
用户输入：
“我想要最近两个 Sprint 的任务趋势与燃尽风险摘要”

系统行为：
1. 拆分任务：趋势分析 + 风险评估 + 摘要输出
2. 调用工具：任务查询 / 燃尽图 / 风险评估
3. 汇总生成摘要
4. 保存会话与结果

## 4. 功能需求
### 4.1 核心功能
- 需求理解与规划
- 工具调用（函数调用）
- 结果汇总与摘要
- 记忆与会话管理

### 4.2 业务能力映射
- `TaskService` → 任务趋势统计
- `BurndownService` → 燃尽图数据
- `RiskService`（或现有风险工具） → 风险评估

### 4.3 API 输出
- 返回 `answer`（摘要）
- 返回 `summary`（风险等级、关键结论）
- 返回 `toolsUsed` 与 `evidence`

## 5. 非功能需求
- 平均响应时间 < 3s
- 支持 Prometheus 监控与报警
- 支持追踪 traceId

## 6. 边界与约束
- 不重构现有业务核心逻辑
- Agent 层独立封装，便于替换或扩展
