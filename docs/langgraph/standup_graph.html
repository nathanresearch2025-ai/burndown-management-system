节点说明

validate_input 校验 sprint_id，初始化状态字段
fetch_core_tools asyncio.gather 并行调用 3 工具，独立捕获异常
validate_data 检测工具失败与数据缺失，决定路由
retry_tools 对失败工具重试 1 次，更新状态
enrich_data burndown 为空时写入占位说明
summarize 调用 LLM，生成 JSON 格式回答
finalize 解析 JSON，失败时 fallback，映射响应字段
early_exit 短路节点：仅查进行中任务，跳过燃尽工具

条件路由

validate_input 后
sprint_id 有值 → fetch_core_tools
sprint_id 缺失 → early_exit
validate_data 后
有工具失败 & retry_count < 1 → retry_tools
burndown 空 & 未 enrich → enrich_data
其他 → summarize
retry_tools 后
burndown 仍缺失 → enrich_data
数据可用 → summarize

状态字段（StandupGraphState）

输入 question · project_id · sprint_id · user_id · trace_id
工具输出 in_progress_raw · burndown_raw · risk_raw
过程控制 tool_errors · missing_fields · retry_count · enriched
最终输出 answer · tools_used · evidence · risk_level
