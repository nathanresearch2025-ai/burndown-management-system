节点说明

Supervisor LLM编排
读取当前状态，LLM 决定 next 字段路由到哪个 Agent，全部完成后输出 FINISH
DataAgent ReActLLM
create_react_agent 内部循环：LLM 自主决定调用哪些工具、调用几次，直到信息足够
AnalystAgent LLM
专项风险分析，输出 risk_level + analysis + key_findings JSON
WriterAgent LLM
整合数据+分析，生成面向团队的中文站会报告 JSON

v1 vs v2 核心差异

工具调用决策 v1：Python 代码写死并行调 3 个工具
v2：DataAgent ReAct 循环，LLM 自主决定
任务分工 v1：单次 LLM 汇总所有输出
v2：3 个专业 Agent 分别负责数据/分析/写作
编排方式 v1：Python 条件路由（确定性）
v2：Supervisor LLM 动态编排
推理可见性 v1：无推理过程
v2：react_messages 记录完整 Thought/Action/Observation

状态流转（StandupV2State）

输入 question · project_id · sprint_id · user_id · trace_id
Supervisor 控制 next: data_agent | analyst_agent | writer_agent | FINISH
DataAgent 输出 data_summary · tools_used · react_messages
AnalystAgent 输出 risk_level · analysis · evidence
WriterAgent 输出 answer · evidence（合并）
