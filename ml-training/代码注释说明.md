## 🔨 第 3 部分：特征工程

### 为什么需要特征工程？

**原理：** 机器学习的本质是"特征工程 + 算法"，好的特征比复杂的算法更重要。

**目标：** 从原始 12 个特征扩展到 27 个特征，提升准确率 +5-10%

---

### 3.1 交互特征（Interaction Features）

```python
# 故事点 × 任务类型
df['story_points_x_type'] = df['story_points'] * df['task_type'].map(
    {'STORY': 1, 'BUG': 2, 'TASK': 3, 'EPIC': 4}
)

# 故事点 × 优先级
df['story_points_x_priority'] = df['story_points'] * df['priority'].map(
    {'LOW': 1, 'MEDIUM': 2, 'HIGH': 3, 'CRITICAL': 4}
)
```

**为什么需要交互特征？**

**问题：** 相同的 5 个故事点，不同类型的任务工时差异很大

| 任务类型 | 5 个故事点的实际工时 | 原因 |
|---------|-------------------|------|
| STORY | 15 小时 | 常规开发 |
| BUG | 10 小时 | 修复问题，范围小 |
| TASK | 20 小时 | 技术任务，复杂度高 |
| EPIC | 40 小时 | 大型史诗，包含多个子任务 |

**解决方案：** 创建交互特征捕获这种非线性关系

**计算示例：**

```python
# 示例 1: STORY, 5 故事点, HIGH 优先级
story_points_x_type = 5 * 1 = 5
story_points_x_priority = 5 * 3 = 15

# 示例 2: EPIC, 5 故事点, CRITICAL 优先级
story_points_x_type = 5 * 4 = 20
story_points_x_priority = 5 * 4 = 20
```

**效果：** 准确率提升 +2-3%

---

### 3.2 比例特征（Ratio Features）

```python
# 描述长度 / 标题长度
df['desc_title_ratio'] = df['description_length'] / (df['title_length'] + 1)

# 是否有描述
df['has_description'] = (df['description_length'] > 0).astype(int)
```

**为什么需要比例特征？**

**观察：** 描述越详细的任务，通常越复杂

| 标题长度 | 描述长度 | 比值 | 工时 | 分析 |
|---------|---------|------|------|------|
| 20 | 0 | 0.0 | 5h | 没描述，可能很简单 |
| 20 | 50 | 2.5 | 15h | 描述适中 |
| 20 | 200 | 10.0 | 30h | 描述很详细，任务复杂 |

**计算示例：**

```python
# 示例 1: 标题 30 字符，描述 150 字符
desc_title_ratio = 150 / (30 + 1) = 4.84

# 示例 2: 标题 50 字符，描述 0 字符
desc_title_ratio = 0 / (50 + 1) = 0.0
has_description = 0  # 没有描述
```

**为什么 +1？** 避免除零错误（当 title_length = 0 时）

**效果：** 准确率提升 +1-2%

### 4.2 类别特征编码

```python
categorical_cols = ['task_type', 'priority']
label_encoders = {}

for col in categorical_cols:
    le = LabelEncoder()
    X[f'{col}_encoded'] = le.fit_transform(X[col])

```

**为什么需要编码？**

机器学习模型只能处理数字，不能直接处理文本。

**编码过程：**

```python
# 原始数据
task_type: ['STORY', 'BUG', 'TASK', 'EPIC', 'STORY', 'BUG']

# 编码后
task_type_encoded: [0, 1, 2, 3, 0, 1]

# 映射关系
STORY → 0
BUG   → 1
TASK  → 2
EPIC  → 3
```

**为什么保存编码器？**

```python
label_encoders[col] = le
```

- 预测新任务时，需要用相同的编码规则
- 例如：新任务是 'STORY'，必须编码为 0（而不是其他数字）

## 📈 第 5 部分：训练基线模型

### 5.1 创建和训练模型

```python
baseline_model = xgb.XGBRegressor(
    max_depth=6,              # 树的最大深度
    learning_rate=0.1,        # 学习率
    n_estimators=200,         # 树的数量
    subsample=0.8,            # 样本采样比例
    colsample_bytree=0.8,     # 特征采样比例
    random_state=42
)

baseline_model.fit(X_train, y_train, verbose=False)
y_pred_baseline = baseline_model.predict(X_test)
```

**参数说明：**

| 参数 | 含义 | 默认值 | 作用 |
|------|------|--------|------|
| max_depth | 树的最大深度 | 6 | 控制模型复杂度，越大越容易过拟合 |
| learning_rate | 学习率 | 0.1 | 每棵树的贡献权重，越小越保守 |
| n_estimators | 树的数量 | 200 | 树越多，模型越强，但训练越慢 |
| subsample | 样本采样比例 | 0.8 | 每棵树随机采样 80% 样本，防止过拟合 |
| colsample_bytree | 特征采样比例 | 0.8 | 每棵树随机采样 80% 特征，防止过拟合 |

**XGBoost 工作原理（简化版）：**

1. 训练第 1 棵树，预测结果
2. 计算预测误差
3. 训练第 2 棵树，专门预测第 1 棵树的误差
4. 重复 200 次（n_estimators=200）
5. 最终预测 = 树1 + 树2 + ... + 树200

---

### 5.2 评估指标

```python
baseline_mae = mean_absolute_error(y_test, y_pred_baseline)
baseline_rmse = np.sqrt(mean_squared_error(y_test, y_pred_baseline))
baseline_r2 = r2_score(y_test, y_pred_baseline)
baseline_accuracy = np.mean(np.abs(y_pred_baseline - y_test) / y_test <= 0.2) * 100
```

#### MAE（平均绝对误差）

```python
MAE = mean(|预测值 - 真实值|)
```

**示例：**

```python
真实值: [10, 20, 30, 40, 50]
预测值: [12, 18, 32, 38, 52]
误差:   [2,  2,  2,  2,  2]
MAE = (2+2+2+2+2) / 5 = 2.0 小时
```

**解释：** 平均每个预测偏差 2 小时

#### RMSE（均方根误差）

```python
RMSE = sqrt(mean((预测值 - 真实值)²))
```

**特点：** 对大误差更敏感

```python
误差:   [2, 2, 2, 2, 10]  # 最后一个误差很大
MAE  = (2+2+2+2+10) / 5 = 3.6
RMSE = sqrt((4+4+4+4+100) / 5) = sqrt(23.2) = 4.8
```

#### R²（决定系数）

```python
R² = 1 - (预测误差 / 基准误差)
```

**取值范围：** 0-1（越接近 1 越好）

**解释：**
- R² = 0.8：模型解释了 80% 的数据方差
- R² = 0.5：模型解释了 50% 的数据方差
- R² = 0.0：模型和随机猜测一样差

#### 准确率（±20%）

```python
accuracy = 预测误差在 ±20% 以内的样本比例
```

**示例：**

```python
真实值: [100, 50, 20]
预测值: [110, 45, 30]
误差率: [10%, 10%, 50%]

前两个在 ±20% 以内，准确率 = 2/3 = 66.7%
```

## 🔍 第 6 部分：Optuna 自动调参

### 6.1 目标函数

```python
def objective(trial):
    params = {
        'max_depth': trial.suggest_int('max_depth', 3, 10),
        'learning_rate': trial.suggest_float('learning_rate', 0.01, 0.3, log=True),
        'n_estimators': trial.suggest_int('n_estimators', 100, 500),
        'subsample': trial.suggest_float('subsample', 0.6, 1.0),
        'colsample_bytree': trial.suggest_float('colsample_bytree', 0.6, 1.0),
        'min_child_weight': trial.suggest_int('min_child_weight', 1, 7),
        'gamma': trial.suggest_float('gamma', 0, 0.5),
        'reg_alpha': trial.suggest_float('reg_alpha', 0, 1),
        'reg_lambda': trial.suggest_float('reg_lambda', 0, 1),
        'random_state': 42
    }

    model = xgb.XGBRegressor(**params)
    model.fit(X_train, y_train, verbose=False)
    y_pred = model.predict(X_test)
    mae = mean_absolute_error(y_test, y_pred)

    return mae
```

**Optuna 工作原理：**

1. **第 1 次试验：** 随机选择参数，训练模型，得到 MAE = 3.5
2. **第 2 次试验：** 根据第 1 次结果，智能选择新参数，得到 MAE = 3.2
3. **第 3 次试验：** 根据前 2 次结果，继续优化，得到 MAE = 2.8
4. **...**
5. **第 50 次试验：** 找到最佳参数，MAE = 2.3

**为什么比网格搜索好？**

| 方法 | 搜索策略 | 试验次数 | 时间 |
|------|---------|---------|------|
| 网格搜索 | 遍历所有组合 | 10^9 = 1,000,000,000 | 几天 |
| Optuna | 贝叶斯优化 | 50 | 10 分钟 |

---

### 6.2 参数搜索空间

| 参数 | 搜索范围 | 说明 |
|------|---------|------|
| max_depth | 3-10 | 树深度，越大越复杂 |
| learning_rate | 0.01-0.3 | 学习率，越小越精细 |
| n_estimators | 100-500 | 树数量，越多越强 |
| subsample | 0.6-1.0 | 样本采样比例 |
| colsample_bytree | 0.6-1.0 | 特征采样比例 |
| min_child_weight | 1-7 | 最小叶子权重 |
| gamma | 0-0.5 | 分裂最小损失 |
| reg_alpha | 0-1 | L1 正则化 |
| reg_lambda | 0-1 | L2 正则化 |
